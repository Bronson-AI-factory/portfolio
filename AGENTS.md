## Tone
Пиши простым русским языком, коротко и по делу.
Без «умных» оборотов ради стиля.

## Project Context
Кратко напоминай: что за продукт, для кого он и какую боль закрывает.
Пример формата: «[продукт] — для [аудитория], которым нужно [ключевая задача/боль] без [страх/барьер]».

## Voice & Style
Пиши от первого лица множественного числа («мы»), если явно не задано другое.
Профлексика (DeFi, on-chain, APY и т.п.) допустима, но каждое непонятное слово можно объяснить в одном простом предложении.

## Docs & Tools
- Перед редактированием кода сначала изучи связанную документацию проекта.
- Перед внедрением новых библиотек читай их актуальную документацию на официальном сайте.
- Перед использованием любого инструмента проверяй официальный источник и актуальную версию.
- Если задача связана с кодом или библиотеками, используй MCP‑сервер Context7 для поиска актуальной документации и примеров, прежде чем что‑то менять.

## Landing Rules
- Каждый блок отвечает на вопрос «что получит человек?».
- Без воды и штампов.
- Текст должен читаться как живой, не как шаблон нейросети.
- Один главный CTA на экран.

## Structure Rules
- На лендинге почти всегда есть: хедер с оффером, блок доверия (факты/цифры/логотипы), «Как это работает», условия/тарифы (если релевантно), FAQ.
- Не убирай блок, который повышает доверие и прозрачность (особенно для финтеха и денег).

## Numbers & Proof
- Каждое сильное обещание по возможности подкрепляй цифрой, фактом или примером.
- Если нет факта — ослабь формулировку или убери.

## Anti-Slop
- Запрещены слова «инновационный», «революционный», «уникальный» без фактов.
- Если фраза звучит как шаблон нейросети — перепиши проще и конкретнее.

## Design / UX Rules
- На мобиле текст должен быть читабелен без зума.
- Не злоупотребляй капслоком и «визуальным шумом».
- На одном экране — не больше одного основного CTA.

## Experiment Rules
- Если проводишь A/B-тест оффера или CTA, фиксируй цель теста и ключевую метрику.
- Эксперименты не отменяют правила Anti-Slop и Numbers & Proof.

## Done Criteria
- Мобильная версия читается без зума.
- Оффер понятен за 3 секунды.
- Есть конкретный следующий шаг (CTA).

## Trigger Rule

Если меняются оффер / CTA / структура страницы — обязательно запускай recurring-задачу Landing Polish из `RECURRING_TASKS.md`.
Любые принятые изменения по лендингу (текст / структура / CTA) должны быть закоммичены и запушены в GitHub.
